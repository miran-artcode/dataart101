# 데이터의 눈으로 그림 읽기 — 디자인 인계 문서 (PRD + 디자인 시스템)

> 서울미술고등학교 미술 감상 학습 플랫폼 · 디자인/프론트 핸드오프
> 최종 업데이트: 2026-06 · 담당 교사: Miran Hwang (황미란)
> 본 문서는 현재까지 제작·반영된 화면과 디자인 규칙의 단일 기준(Source of Truth)입니다.

---

## 1. 제품 개요

학생이 그림을 **'느낌'이 아니라 데이터로** 읽도록 돕는 미술 감상 학습 플랫폼.
작품의 **색·구도·객체·소리·맥락**을 데이터로 추출해 새로운 언어로 번역하고,
그 데이터를 재료 삼아 **나만의 미디어아트로 재창조**하며,
친구의 작품을 **데이터를 근거로 비평**한다. AI는 정답이 아니라 **더 좋은 질문**을 건넨다.

**핵심 원칙**
- 정답을 주지 않는다 → 선택에 이유를 붙이게 한다.
- 색은 작품(데이터)에서만 나온다 → UI 크롬은 무채색/뉴트럴.
- 읽기(분석) → 재창조(작업실) → 비평(전시)의 순환.

## 3. 정보 구조(IA) · 플로우

```
(랜딩) 메인페이지
   └─▶ 허브(앱 진입점)
         ├─▶ 분석 도구 ─ 그림분석 [구현] / 소리·내삶·사회 [예정]
         │       └─▶ 작업실(재창조)
         │               └─▶ 전시(출품·비평)
         ├─▶ 전시(3D 전시실)
         ├─▶ AI 감상 코치 (전 단계 보조)
         └─▶ (교사) 교사 대시보드
```
- 학생 메인 동선: 허브 → 그림분석 → 작업실 → 전시
- AI 코치는 모든 단계에서 호출 가능한 보조 레이어.

## 4. 디자인 시스템

### 4.1 컨셉 규칙 (가장 중요)
- **두 컨셉을 한 화면에 섞지 않는다.** 한 페이지는 라이트이거나 다크 중 하나로 통일.
- **하이브리드 테마** (한 화면 = 한 컨셉 유지):
  - **라이트** = 메인 랜딩 · 허브 · 교사 대시보드 · 로그인 · 내 작업 (진입·학습·관리, 가독성 우선).
  - **다크(몰입)** = 그림분석 · 소리 · 내 삶 · 사회 분석 · 작업실 · 전시 (작품·데이터가 어둠 위에서 빛나는 뷰어·제작·전시).
- 타이포그래피 자체가 디자인 요소다 (대형 디스플레이 활용).
- **금지**: 따뜻한 색(코랄·골드·테라코타) / '빨노초파' 무지개식 다색 / 한 줄에 비슷한 색·동일 비율 반복.

### 4.3 타이포그래피
- **본문/UI**: Pretendard (system fallback).
- **디스플레이/키커**: Orbitron (영문 대문자) — 페이지 키커(HUB·ANALYSIS·STUDIO·GALLERY·AI COACH·DASHBOARD), 메인 헤드라인, 배지/백월 텍스트.
- 메인 헤드라인 `READ ART / WITH DATA` = **채움 + 외곽선(text-stroke)** 대비.
- 키커: 12px / 700 / letter-spacing 0.2em / 색 라이트 `#3A4F7A`, 다크 `#93A4E8`(또는 틸).

### 4.4 간격·모양·그림자
- 컨테이너 `max-width:1180px`, 좌우 패딩 18px.
- radius: 카드 20–24px, pill 999px, 칩 999px.
- 그림자: soft 1단계 (`0 24~50px ... -38~-44px rgba(...)`).
- 8 기반 간격.

### 4.5 공통 컴포넌트
- **내비**: 스티키 플로팅 바. 라이트=흰 pill, 다크=글래스(`rgba(255,255,255,0.04)` + blur). 로고 = 잉크 칩 + 렌즈(눈) 마크.
- **회전 크레딧 배지**(메인 히어로): 원형 회전 텍스트 `© MIRAN HWANG · SEOUL ARTS HIGH SCHOOL · ART TEACHER ·` + 중앙 렌즈 마크 + `2025`.
- **풋터**(전 페이지 공통, 다크 `#0A0E26`): 로고 + `© 2025 Miran Hwang · Seoul Arts High School · Art Teacher` + 내비 pill 4개 + 블루·틸 글로우.
- **버튼**: 1차 = 잉크(라이트)/블루 `#2F6BE0`·화이트(다크), 2차 = ghost(보더).
- **데이터 위젯**: 색 군집 막대(가변 비율) · 렌즈 바 · 객체 박스 · 파형 · 산점도 · 헥사곤 렌즈.

## 5. 페이지별 명세 (`*.dc.html`)

> 모든 파일은 Design Component(`.dc.html`). 같은 폴더의 `support.js` 필요. 브라우저에서 바로 열림.

### 5.1 메인페이지.dc.html — 라이트 랜딩
- 타이포 히어로 `READ ART / WITH DATA`(채움+외곽선) · 회전 크레딧 배지 · 키커 `DATA × ART`.
- About(프로젝트 소개 + Data Portrait 비주얼) · 분석 도구 4종 미리보기 · 마감 CTA · 풋터.
- Tweaks(props): `showBadge`(bool) · `animateBadge`(bool) · `glow`(off/soft/strong).

### 5.2 허브.dc.html — 다크 · 앱 진입점
- 인사 헤더(HUB 키커) · 분석 도구 4종 카드(그림 활성) · '이어서 작업하기'(가변 비율 썸네일) · 전시/AI코치 프로모.

### 5.3 그림분석.dc.html — 다크 · 분석 도구 [기능 핵심, 보존]
- 작품 슬롯(`image-slot.js`, 드래그 업로드) · 한눈에(객체·명도·채도) · **색 군집 6**(비율 막대+리스트) · **7가지 분석 렌즈**(바) · **객체 감지**(그리드+바운딩 박스+칩) · 다음 단계 CTA(→작업실).

### 5.4 작업실.dc.html — 다크 · 재창조
- 좌: 분석 데이터(재료) · 중앙: **제너러티브 캔버스**(입자/선/격자 모드, 실시간) · 우: 슬라이더(밀도·흐름·크기) + 입력 연결(데이터→형태 매핑).

### 5.5 전시.dc.html — 다크 · 3D 전시실
- **CSS 3D 워크스루 전시실**: 바닥/천장/좌우 벽/백월, 양벽에 스포트라이트 받은 학생 작품(데이터 색면) + 라벨 플레이트.
- `←/→`로 관람 위치 이동(카메라 `translateZ`), 작품 클릭 → 그림분석.
- 하단: 오늘의 비평(데이터 근거 대화).

### 5.6 AI코치.dc.html — 다크 · 감상 코치
- 좌: 작품 컨텍스트 + 데이터 칩 · 우: 채팅(질문형 응답, 제안 답변 칩, 입력). 정답 대신 질문.

### 5.7 교사대시보드.dc.html — 다크 · 교사
- 통계 4(참여·제출·분석·대기) · 학생 진행 로스터(단계 바·작품 썸네일·피드백) · 학급 색 경향 · 검토 대기.

## 7. 변경 이력 (이번 작업에서 반영)

1. 메인페이지 신규: 타이포 히어로(채움+외곽선) + 회전 크레딧 배지(Miran Hwang).
2. 팔레트 전면 교체: 어스톤(드랍) → 쿨 톤 램프(감청·슬레이트·세이지·연두). 무지개/웜톤 제거.
3. 색 막대·썸네일 전부 **가변 비율**로(동일 폭 반복 제거).
4. 전 페이지에 Orbitron 영문 키커 + 다크 풋터(© 크레딧) 적용.
5. **컨셉 분리**: 메인=라이트 / 앱=다크. 메인의 다크 육각 섹션 제거(혼합 제거).
6. 그림분석·허브·작업실·전시·AI코치·교사대시보드 = **다크로 통일**.
7. 전시 = **3D 전시실**로 재구성.
8. 글로우 코랄(웜) 제거 → 블루·바이올렛·틸.

```
├─ 메인페이지.dc.html        (라이트 랜딩)
```

- **커밋(GitHub)**:
  ```bash
  cd your-repo
  git checkout -b design/dark-system
  git add .
  git commit -m "데이터의 눈으로 그림 읽기 — 라이트 랜딩 + 다크 앱"
  ```
